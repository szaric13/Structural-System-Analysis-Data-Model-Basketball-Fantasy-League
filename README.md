# Structural-System-Analysis-Data-Model-Basketball-Fantasy-League

Structural System Analysis & Data Model – Basketball Fantasy League
Project Description

This project focuses on the structural system analysis and data modeling of a Basketball Fantasy League Information System.

The goal of the analysis was to model and design a centralized digital system that supports user management, team creation, player selection, scoring automation, league management, and ranking calculation.

The system was documented using structured analysis techniques including context diagrams, data flow diagrams (DFD) with multiple decomposition levels, data dictionary, mini-specifications, and an entity-relationship data model.

Key Components

Context Diagram – Identified at least four external entities interacting with the system (users, administrators, external data sources, etc.).

Data Flow Diagrams (DFD) –

Level I with at least five core processes

Level II with further decomposition of each process into 2–3 sub-processes

Data Dictionary – Decomposition of key data flows and definition of system data elements.

Mini-Specifications – Defined for seven primitive processes describing system logic and behavior.

Data Model –

Definition of entities such as Users, Teams, Players, Leagues, Matches, Statistics, and Rankings

Attributes, primary and foreign keys

Relationships and cardinalities

Represented through an ER diagram
